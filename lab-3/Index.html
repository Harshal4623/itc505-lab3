Lab 3: Serving a Custom Font

Web browsers usually display text using fonts that are already installed on a userâ€™s computer. This can be limiting if a website needs a specific or unique style.

To solve this, I served my own font directly from the web server using the CSS @font-face rule. This rule allows a webpage to define a custom font and tell the browser exactly where the font file is located.

When this page loads, the browser first downloads the HTML file and then the CSS file. The CSS file points to the font file stored in the fonts folder. The browser then requests that font file just like it would request an image or stylesheet.

Once the font is downloaded, the browser uses it to render the text on the page. Because this font is unusual and not commonly installed, it is very easy to see that the custom font has been loaded successfully.
